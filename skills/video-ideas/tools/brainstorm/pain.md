# Pain

## Purpose

Partly **Radical Acceptance** (Stutz): list the pain. Don't tidy it into a redemption arc. List out all the hard stuff happening to you; list all the things that hurt; just list them; just write them down. Declare these are pains.

They could be really bad, physically painful, just complaints; things you were actually affected by; or even hard things that happened around you. By processing this, by writing them down and listing them, some ideas could come out of it. Not just how you persisted or overcame. Just stating the pain clearly could help others.

## Inputs

Life / work / domain context. Optional time window.

## Mode

`interactive`. Countdown to 20 pains. Optional light second pass for video angles that stay in acknowledgment (not obligatory triumph).

## Steps

1. Ask for life / work / domain context (optional time window).
2. Prompt: list all the pain: physical, complaints, things that hit you, hard things around you.
3. Declare them. Keep listing until 20 or the user signals done.
4. Optional second pass (only if user wants): ideas that help others sit with / name similar pain, not only "how I beat it."

## Stop

Flat pain list is captured. User picks whether to map to video angles.

## Output

1. Flat list of pains (blunt, specific)
2. Optional: ideas that help others sit with / name similar pain, not only "how I beat it"

## Anti-patterns

- Do not force a persistence / overcome / silver-lining frame
- Do not skip listing to jump to "content ideas" before the pain is named
- Do not minimize ("that's not so bad"). Declare it pain
- Do not conflate with Worst Critic, CANT, or Awful. This is what hurt / is wrong / happened
